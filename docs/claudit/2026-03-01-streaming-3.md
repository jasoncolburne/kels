## High Priority

### ~~1. Anchor checking no longer excludes divergent events ÔÇö security regression~~ RESOLVED

Added `diverged_at_serial` guard in `verify_generation`: anchors at or after the divergence serial are not recorded in `anchored_saids`. Pre-divergence anchors remain trusted. Regression tests: `test_anchors_on_divergent_branches_excluded`, `test_anchors_after_divergence_excluded_multi_serial`.

### ~~2. `completed_verification` does not fail secure on `max_pages` exhaustion~~ RESOLVED

Added `exhausted` flag tracking. When the loop exits due to `max_pages` exhaustion (not source depletion), returns `Err(KelsError::InvalidKel(...))` instead of a partial `Verification`. Regression tests: `test_max_pages_limit_fails_secure`, `test_max_pages_exact_boundary_succeeds`, `test_max_pages_one_over_boundary_fails`.

### ~~3. Gossip sync only fetches the first page of delta events~~ RESOLVED

Added `KelsClient::fetch_all_key_events()` which paginates through all pages with a `max_pages` guard (fails if exceeded). Replaced all 5 `fetch_key_events` call sites in gossip `handle_announcement` with `fetch_all_key_events`.

### ~~4. Unbounded pagination loops in divergence scan helpers~~ RESOLVED

Added `max_verification_pages()` guard to all 5 scan loops in `handlers.rs`: `recovery_revealed_in_divergent_events`, `scan_divergent_events`, `verify_chain_before_serial`, `non_contest_recovery_revealed_since`, and the divergent event collection loop in `handle_overlap_submission`. All fail secure with an error on exhaustion.

## Medium Priority

### ~~5. Every submission pays full `completed_verification` cost~~ RESOLVED

Added comment block above `completed_verification` in the submit handler explaining why re-verification is required on every submission (DB cannot be trusted), why caching `Verification` tokens is unsafe (stale token + tampered DB = accepting invalid events), and a TODO for optimization via batch signature verification and eager/parallel page loading.

### ~~6. `apply_submit_key_events` DB persist errors logged at `debug` level~~ RESOLVED

Added `DuplicateRecord` variant to `StorageError` (in `verifiable-storage-rs`) with PostgreSQL unique_violation (23505) detection in `map_sqlx_insert_error` helper. State machine now matches on the variant: `DuplicateRecord` logs at `debug` (expected on Raft replay), all other errors log at `error` with a warning about missing events.

### ~~7. Cache hit in `get_kel` deserializes and re-serializes unnecessarily~~ RESOLVED

Changed `get_kel` to use `build_page_bytes` for the cache hit path, returning a raw `Response` with `application/json` content type. Eliminates the `Vec<u8>` Ôćĺ `Vec<SignedKeyEvent>` Ôćĺ `SignedKeyEventPage` Ôćĺ JSON round-trip. Return type changed from `Json<SignedKeyEventPage>` to `Response`; non-cache paths use `.into_response()`.

### ~~8. `raft_event_count` computed from `tip.serial + 1` assumes contiguous zero-based serials~~ RESOLVED

Added `event_count: usize` field to `Verification` and `KelVerifier`. `KelVerifier` tracks the count: initialized to 0 in `new()` and `from_branch_tip()`, resumed from `ctx.event_count()` in `resume()`, incremented by `events.len()` in `verify_page()`, passed through `into_verification()`. Replaced all 4 call sites using `tip.serial + 1` (sync.rs, handlers.rs x2, kels-cli) with `ctx.event_count()`.

### ~~9. No integration tests for divergence/recovery/contest in refactored handlers~~ RESOLVED

Added 4 integration tests in `services/kels/tests/integration_tests.rs`:
- `test_divergence_creation` ÔÇö submits conflicting interactions at same serial, verifies two events at serial 3
- `test_recovery_from_divergence` ÔÇö creates divergence, submits recovery (rec + rot), verifies KEL recovered and subsequent appends work
- `test_contest_freezes_kel` ÔÇö owner rotates recovery (revealing key), adversary submits contest, verifies KEL frozen and rejects further submissions
- `test_overlap_submission_creates_divergence` ÔÇö submits overlapping events (some already on server) with divergent tip, verifies dedup + divergence detection

## Low Priority

### ~~10. Import ordering violation in `handlers.rs`~~ RESOLVED

Reordered imports in `services/kels/src/handlers.rs`: `std` first (group 1), then external crates (group 2), then local `crate::` imports (group 3), with blank lines between groups.

### ~~11. `get_effective_said` endpoint returns `serde_json::Value` instead of typed struct~~ RESOLVED

Added `EffectiveSaidResponse { said: String }` struct in `lib/kels/src/types/events.rs`, exported from `lib.rs`. Handler now returns `Json<EffectiveSaidResponse>` instead of `Json<serde_json::Value>`.

### ~~12. Silent fallback on malformed `KELS_MAX_VERIFICATION_PAGES` env var~~ RESOLVED

Added `use tracing::warn;` to `lib/kels/src/lib.rs`. `max_verification_pages()` now logs a warning when the env var is set but unparseable, before falling back to the default.

### ~~13. Variable shadowing of `ctx` in `show_identity_status`~~ RESOLVED

Renamed the local `ctx: Option<Verification>` to `verification` in `show_identity_status`, updated the one usage site (`ctx.as_ref()` Ôćĺ `verification.as_ref()`).

### ~~14. `transfer_key_events` composite SAID cursor fragility~~ RESOLVED

Added doc comment on `transfer_key_events` explaining: composite SAID cursor after divergence, staleness when source KEL state changes between fetches, and guidance for callers to retry from scratch on `EventNotFound`.

### ~~15. `StoreKelSource::fetch_page` loads entire KEL into memory for `since`-based pagination~~ RESOLVED

Replaced `StoreKelSource` doc comment with explicit warning that `fetch_page` with `since` loads the entire KEL into memory (O(N)), only suitable for dev-tools/CLI, not production workloads.

### Bonus: `KelsError::KeyNotFound` Ôćĺ `KelsError::EventNotFound`

Renamed across ~30 call sites to avoid confusion with cryptographic keys. `HsmError::KeyNotFound` (which IS about cryptographic keys) left untouched. Error message updated from "Key not found" to "Event not found". Doc references in `gossip.md` and `serving.rs` updated.

## Positive Observations

- **Verification invariant well enforced:** `Verification` has no public constructor ÔÇö only obtainable via `KelVerifier::into_verification()`. Type system prevents consuming unverified KEL data.
- **Clean API design:** `SignedKeyEventPage`, `PagedKelSource`/`PagedKelSink`, `KelServer` trait provide clear abstractions for pagination and streaming.
- **Deterministic ordering maintained:** All paginated queries use `ORDER BY serial ASC, CASE kind ... END ASC, said ASC` consistently.
- **Good test coverage for new paths:** Multi-page verification, anchor checking across pages, divergence at page boundaries, transfer with held-back events, partition-for-submission ÔÇö all well tested.
- **`compute_prefix_effective_said` uses efficient SQL:** `NOT EXISTS` correlated subquery with proper index (`idx_kels_key_events_prefix_previous_idx`) avoids loading all events.
- **Recovery path double-checks:** Belt-and-suspenders approach in `handle_divergent_submission` with both `recovery_revealed_in_divergent_events` and `scan_divergent_events` checking adversary recovery independently.
