## High Priority

### ~~1. `fetch_peers()` has a wasteful, side-effectful readiness probe that adds latency to every caller~~ ‚Äî RESOLVED

**File:** `lib/kels/src/client/registry.rs:157-175`

~~The function fires parallel HTTP readiness checks against every peer's KELS service and discards the results. Adds up to 2 seconds of latency to every `fetch_peers()` call. **Resolution:** Removed readiness probing from `fetch_peers()`. Added explicit readiness probing to the gossip service startup (`services/gossip/src/lib.rs`) ‚Äî the only caller that needs the delay. Probes only active peers.

**File:** `lib/kels/src/client/kels.rs:111-114`

~~The method builds a new `reqwest::Client` with a 2-second hardcoded timeout, allocating a new connection pool each time.~~

**Resolution:** Now uses `self.client` with a per-request 2-second timeout via `.timeout(Duration::from_secs(2))`.

## Positive Observations

- **Types reorganization is clean.** The subdirectory structure (`kel/`, `sad/`, `federation/`, `gossip/`) groups types by domain while preserving all re-exports so downstream code doesn't change. No types were lost in the move.

- **`wait_for_divergence_convergence` replaces fragile sleep.** The SAD test now polls for both nodes to agree on divergent state rather than sleeping a fixed duration, making the test deterministic.

- **Readiness probing correctly scoped to gossip startup.** After identifying that removing the implicit readiness delay from `fetch_peers()` broke gossip mesh formation, the probing was placed explicitly in the gossip service startup ‚Äî the only consumer that needs it ‚Äî rather than penalizing all callers.

- **SadAnnouncement correctly scoped to gossip service.** Moving it from the shared library to the only service that uses it is good encapsulation. The type definition is identical so wire compatibility is preserved.

- **Import ordering follows CLAUDE.md conventions.** All checked files use the 3-group pattern (std, external, local) consistently.

- **`NodeInfo` removal is thorough.** All 9 files that referenced `NodeInfo`/`NodeStatus`/`NodeType` were updated, including FFI, Swift client, CLI, and tests. No stale references remain.
