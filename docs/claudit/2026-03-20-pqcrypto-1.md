## Positive Observations

- **Removal of ECDH and public key from handshake.** The old three-DH pattern (ee + se + es) required transmitting public keys over the wire and performing ECDH via the HSM service. The new ML-KEM approach is simpler (single KEM operation), eliminates the static-ephemeral DH (removing the need for the `ecdh` endpoint and HSM ECDH capability), and the verifier now fetches the peer's public key from their KEL directly rather than trusting what the peer sends. This is a significant simplification and security improvement.

- **Post-quantum forward secrecy via ML-KEM.** Sessions are now protected by ML-KEM-768 or ML-KEM-1024, providing quantum-resistant key encapsulation. The shared secret feeds into blake3 KDF for AES-GCM-256 session keys. This is a well-designed cryptographic upgrade path.

- **Auto-negotiation of KEM algorithm with fail-secure default.** The `requires_kem_1024` flag starts at `true` (use ML-KEM-1024 until the federation's algorithms are known) and is only relaxed after verifying that no peer uses ML-DSA-87. This correctly follows the fail-secure principle.

- **HSM service elimination.** Replacing the HTTP-based HSM service with direct PKCS#11 via `cryptoki` removes a network hop, eliminates a service to deploy/secure, and reduces the attack surface. The mock HSM (`kels-mock-hsm`) provides a clean development-time substitute.

- **Signature label rationalization.** Changing from `public_key` to `label` ("signing"/"recovery") in `EventSignature`/`KeyEventSignature` cleanly decouples the signature lookup mechanism from the key identity. The verifier still cryptographically verifies against the expected key from the KEL — security is preserved while enabling algorithm-agnostic signature handling.

- **Comprehensive test coverage.** New tests cover ML-DSA-65 key generation, signing, save/load roundtrips, mixed algorithm configurations (P-256 signing + ML-DSA-65 recovery), and algorithm upgrade paths. The existing test suite was thoroughly updated to pass algorithm parameters.
