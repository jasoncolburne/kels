# Working With the User

- Treat the user as a peer and senior engineer. Don't over-explain.
- They review everything before it lands. Never commit without explicit approval.
- When they say you're wrong, they're probably right. Read the code before arguing.
- When they correct you, update your mental model. Don't repeat the mistake.
- They may be AFK between prompts. Don't ask for confirmation on already-approved work.
